# product.md 記載ルール

ビジネス文脈・目的・ターゲットユーザー・スコープを記載する。

## フォーマット

```
## プロジェクト概要
1〜3文でプロジェクトの目的を説明する

## 解決する問題・背景
現状の課題と、なぜそれを解決する必要があるか

## ターゲットユーザー
誰が・どのくらいの頻度で使うか

## 今回のスコープ
### やること
- 〜

### やらないこと（スコープ外）
- 〜

## 成功基準（KPI・完了条件）
- 定量的な指標で記載する（「速くする」ではなく「3秒以内に応答する」）

## 制約事項
- 予算・期限・使用禁止技術・法的制約など

## ステークホルダー
- 役割: 氏名 / チーム名
```

## ルール
- 成功基準は必ず定量的に記載する（曖昧な表現禁止）
- 「やらないこと」を明示し、スコープ外の機能をAIが勝手に実装しないようにする
