# requirements.md 記載ルール

EARS記法を使い、機能要件・非機能要件を記載する。

## 要件IDの形式
- 機能要件: `REQ-001`, `REQ-002`, ...
- 非機能要件: `NFR-001`, `NFR-002`, ...

## EARS記法の5パターン

| パターン | 記述形式 |
|----------|----------|
| 常時適用型 | `システムは常に〜しなければならない` |
| イベント駆動型 | `〜したとき、システムは〜しなければならない` |
| 状態依存型 | `〜の間、システムは〜しなければならない` |
| オプション型 | `〜の場合のみ、システムは〜しなければならない` |
| 異常系 | `〜の場合、システムは〜しなければならない` |

## フォーマット

```
## 機能要件

### [機能グループ名]

| ID | パターン | 要件 |
|----|----------|------|
| REQ-001 | イベント駆動型 | 〜したとき、システムは〜しなければならない |
| REQ-002 | 異常系 | 〜の場合、システムは〜しなければならない |

## 非機能要件

| ID | 種別 | 要件 |
|----|------|------|
| NFR-001 | パフォーマンス | システムは常に、〜を〜以内に完了しなければならない |
| NFR-002 | セキュリティ | システムは常に、〜を〜しなければならない |

## スコープ外（やらないこと）
- 〜機能は今回の対象外
```

## ルール
- 正常系だけでなく必ず異常系要件も含めること
- 曖昧な表現（「適切に」「速く」「きれいに」）を禁止し、定量的な基準に変換する
- 各要件は独立して検証可能な粒度にする
