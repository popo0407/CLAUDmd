# structure.md 記載ルール

プロジェクトのフォルダ・ファイル構成と各要素の役割を記載する。

## フォーマット

```
## 目的
このプロジェクトが何のために存在するか（1〜3文）

## ファイル・フォルダ構造
project-root/
├── folder/          # 役割コメント
│   └── file.py      # 役割コメント
└── documents/       # ドキュメント類

## 不明点・確認が必要な事項
- 解析できなかったファイルや用途不明の要素を列挙
```

## ルール
- フォルダは末尾に `/` を付ける
- 各ファイル・フォルダに `# 役割コメント` を付ける
- 解析できない要素は「不明点」セクションに記載し、推測で埋めない
