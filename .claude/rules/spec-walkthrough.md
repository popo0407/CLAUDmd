# walkthrough.md 記載ルール

実装完了後に生成する Antigravity スタイルの実装記録。上限は **150行**（超える場合は要件カバレッジ表を `walkthrough_coverage.md` に分離）。

## フォーマット

```
## 実装サマリー
- 実施日時: YYYY-MM-DD
- 対象機能: （product.md の今回スコープを転記）
- 総タスク数: X件 / 完了: X件 / スキップ: X件

---

## 実装ログ

### TASK-001: [タスク名]
- ステータス: ✅ 完了 / ⚠️ 一部対応 / ❌ スキップ
- 実施内容: 何をどのように実装したか（2〜5文）
- 変更ファイル: `path/to/file.py`
- テスト結果: PASS / FAIL（テストコマンドと出力サマリー）
- 特記事項: 設計書から変更した点・理由（なければ「なし」）

---

## 要件カバレッジ

| 要件ID | 要件概要 | 対応タスク | 検証結果 |
|--------|----------|------------|----------|
| REQ-001 | 〜したとき、〜しなければならない | TASK-001 | ✅ PASS |

全要件カバレッジ: X / X

---

## 設計変更・追加決定の記録（ADR補足）

### 変更点1: [タイトル]
- 背景: 実装中に判明した事実や制約
- 変更内容: design.md から何をどう変えたか
- 理由: なぜ変更が必要だったか
- 影響: どのファイル・要件に影響するか

---

## 最終ファイル構成
（新規作成・変更されたファイルのツリー）

---

## 次のステップ（推奨）
1. design.md に ADR 補足を追記する
2. system_tree.md を最新状態に更新する
```

## ルール
- テスト結果は「実行した事実」のみ記載し、推測を書かない
- 設計から変えた箇所は必ず理由とともに記録する
- 未対応の要件がある場合は次のステップに追記する
