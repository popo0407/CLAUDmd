# 記載ルール
以下の形式で内容を記載すること

## 1. ファイル・フォルダ構造

プロジェクト内のフォルダ・ファイルの階層構造を視覚的に示す。
どこに何があるかを一覧で把握できるようにするためのもの。

ディレクトリツリー形式で記載する。

```
project-root/            # プロジェクトルート
├── frontend/            # フロントエンド（HTML/CSS/JS）
│   ├── index.html       # エントリポイント
│   ├── css/
│   │   └── style.css    # スタイルシート
│   └── js/
│       └── app.js       # メインスクリプト
├── backend/             # バックエンド
│   ├── lambda/
│   │   └── handler.py   # Lambda関数（APIの処理本体）
│   └── requirements.txt # Pythonライブラリ定義
└── documents/           # ドキュメント類
    └── system_tree.md   # システム構成図（本ファイル）
```

### ルール
- フォルダは末尾に `/` を付ける
- ファイルには `# 役割コメント` を付ける
- 深さは実態に合わせて記載する
- 新規作成・削除・移動があった際は必ず更新する
- **`.` で始まるファイル・フォルダ（`.claude/`、`.claudeignore` 等）は記載しない**
- **`CLAUDE.md`、`CLAUD.txt` 等のAI設定ファイルは記載しない**

---

## 2. データフロー

矢印（`-->`）を使い、処理の流れをテキストで記載する。

```
[ユーザー]
    --> [CloudFront] (静的コンテンツ配信)
    --> [API Gateway] (APIリクエスト)
    --> [Lambda] (ビジネスロジック)
    --> [S3 / DynamoDB] (データ永続化)
```

### ルール
- 各ノードは `[名前]` で囲む
- ノードの後ろに `(説明)` で役割を補足できる
- 分岐がある場合は複数行に分けて記載する
- フロー名（例：`### 登録フロー`）を見出しで区切る

---

## 3. 更新タイミング

以下のタイミングで `documents/system_tree.md` を更新すること。

- 新しいフォルダ・ファイルを作成したとき
- 既存のファイルを削除・移動したとき
- AWSリソースを追加・変更したとき
- 処理フローが変わったとき
