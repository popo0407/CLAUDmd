# Elite Frontend UX — Dashboard Patterns

> **前提:** `elite-frontend-ux` (core) と合わせて使うこと。

---

## レイアウト構造

```
┌─────────────────────────────────────────┐
│ Top Bar (56-64px): Logo, Search, User   │
├──────────┬──────────────────────────────┤
│ Sidebar  │  Main Content               │
│ 240-280px│  breadcrumbs (深いnavのみ)   │
│ 折りたたみ│                             │
│ 64-80px  │  Cards / Data / Forms       │
└──────────┴──────────────────────────────┘
```

**ナビゲーション選択:**
| 状況 | パターン |
|---|---|
| 10+セクション | 折りたたみサイドバー |
| 3-6セクション | トップナビ |
| セカンダリナビ | タブ（最大6つ） |
| 深い階層 | パンくずリスト |

---

## コンテンツ優先順位

1. **価値ファースト指標:** 「4時間節約しました」> 生数値
2. **次のアクション:** ユーザーが次に何をすべきか示す
3. **段階的開示:** サマリー → 詳細はオンデマンド
4. **ロールベース:** ペルソナごとに必要なデータが異なる

---

## データ可視化

- 色のセマンティクス: 赤=マイナス、緑=プラス（色覚特性向けにパターン/アイコンも併用）
- 凡例は必須
- 軸ラベルは必須
- 長いラベルはツールチップで展開

---

## 空状態

```jsx
// ✅ 行動を促す空状態
<EmptyState
  icon={<InboxIcon />}
  title="メッセージはまだありません"
  description="メッセージを受信するとここに表示されます。"
  action={<Button>メッセージを作成</Button>}
/>

// ❌ 何も伝わらない
<p>データがありません</p>
```

---

## 設定ページ

- 複雑な設定はバケット + サイドパネルレイアウト
- 破壊的アクションはページ最下部の「Danger Zone」にまとめる
- 削除確認: テキスト入力を要求し、ボタンラベルを具体的に（「削除」ではなく「アカウントを削除」）

---

## Toast / 通知のタイミング

- デフォルト: 4-5秒
- アクセシビリティ最低: 6秒
- 計算式: 500ms × 単語数 + 3秒
- 必ず閉じるボタンを付ける
