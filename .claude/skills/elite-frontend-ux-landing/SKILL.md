# Elite Frontend UX — Landing Page Patterns

> **前提:** `elite-frontend-ux` (core) と合わせて使うこと。

---

## ファーストビュー（必須要素）

ビューポート内に必ず収める：
1. 明確な見出し（5-10語）
2. サブ見出し（価値提案）
3. 単一のプライマリCTA
4. ビジュアル要素（ヒーロー画像・イラスト・プロダクト画面）

---

## セクション構成

```
1. Hero（見出し + CTA + ビジュアル）
2. Social Proof（ロゴバー、証言の抜粋）
3. Problem / Solution
4. Features / Benefits（最大3-4個）
5. 詳細な証言
6. Pricing（必要な場合）
7. FAQ
8. Final CTA
9. Footer
```

---

## CTAボタン

- 最小高さ: 44px、padding = フォントサイズの2倍
- 色: 高コントラスト、暖色は緊急感を生む
- 文言: 動作動詞 + 一人称（「無料で始める」 > 「登録」）
- 長さ: 2-5語以内
- 1ビューポートにプライマリCTAは1つ

---

## Social Proof の配置

- ロゴバー: Heroの直後
- 証言: 異議が生まれるポイントの近く
- 統計数字: 料金表の近く
- トラストバッジ: フォーム・チェックアウトの近く

---

## 料金表

- 最大3-4プラン（それ以上は選択麻痺）
- 推奨プランをハイライト（「最も人気」）
- 年払い/月払い切り替えで節約額を表示
- チェックマークで機能を一覧表示
- 各プランにCTAボタン

---

## フォーム最適化

- 1カラムレイアウト（複数カラムよりエラー率が120%低下）
- フィールド数を最小化（4フィールド vs 11 → 転換率120%向上）
- 電話番号は必須でなければ聞かない（58%が離脱）
- ラベルはinputの上に配置（placeholder代替ではなく）
- バリデーションはblur時、入力中は行わない
