## 基本方針
- プロジェクト名は[プロジェクト名]とする。
- 環境は[dev]とする。
- AWSプロファイルは[AWSプロファイル]とする。
- AWS CLIを使う前に認証状況を確認する。認証用コマンドは以下。

- フォールバック処理は原則として実装しない。必要と判断した場合は、理由を説明のうえユーザーに確認してから実装する。
- 不明確な点・曖昧な仕様がある場合は、推測で進めず必ずユーザーに確認する。
- 新しいトラブル（エラー・バグ・設定問題など）を解決した場合は、スキルの記録に記載の形式で記録する。

## スキルの記録
- `.claude/skills/[スキル名]/SKILL.md` を作成

```
---
name: [スキル名]
description: [「いつ」「何を」「どうするスキルか」を説明する一文、どういう時に呼び出してほしいかを重点的]
# Instructions
[どう処理すべきか300行以内で記載]
---
```
